
# Rule: Environmental Audit Protocol (进场验货协议 - 地理切换)

## 1. When to Use (何时使用)

- 写作内容正文时
- 涉及地理位置切换、进入新场景时
- 描写城市、建筑、自然景观时
- 改进环境描写段落时

## 2. Rule instruction (使用说明)

**[Core Logic]** 拒绝“导游式”的风景描写。马六进入新地图，不是来旅游的，是来**“验货” (Site Inspection)** 的。

### 1. 拒绝全景，寻找“破绽” (No Panorama, Find the Cracks)

*   **原则：** 宏大的景象（城墙、宫殿、山川）通常是用来骗人的。必须把视线聚焦在**“维护不到位”**或者**“暴露真实熵增”**的工业细节上。
*   **操作：**
    *   别看城楼多高，看城墙根的积雪是不是发黑，上面是不是撒了防滑的煤灰。
    *   别看宫殿多金碧辉煌，看地砖缝里是不是长了暗绿色的苔藓，或者柱子的漆皮是不是在起泡。
*   **Why:** 完美是假的，破绽才是真的。破绽意味着“麻烦”、“隐患”和“这是个真实的世界”。

### 2. 生理感官先行 (Physiological First)

*   **原则：** 视觉是最后起作用的感官，因为它有距离感。必须先让**“身体”**直接撞上环境。
*   **操作顺序：** 触觉/嗅觉/听觉（入侵性强） -> 视觉（入侵性弱）。
*   **关键词库：** 湿热、黏腻、咸腥、腐烂、砂砾感、风湿痛、各种炎症诱发。
*   **Example:**
    *   *Bad:* “大梁城很繁华，护城河宽阔。”
    *   *Good:* “一进大梁城，空气里的湿度直接把衣服贴在了后背上（触觉）。护城河散发着一股洗不掉的脂粉味混着泔水味（嗅觉），熏得我偏头痛都犯了（生理反应）。”

### 3. 给环境贴价格标签 (The Pricing Tag)

*   **原则：** 对于项目经理来说，环境不仅仅是背景，更是**成本 (Cost)**。每一个环境特征，都要转化为对“任务难度”或“经费损耗”的评估。
*   **转化公式：**
    *   **繁华** = 物价虚高，住宿超标，报销单很难填。
    *   **荒凉** = 补给困难，鞋底磨损率上升，痛风发作概率+50%。
    *   **险峻/雄伟** = 膝盖杀手，运输成本指数级上升，加班预定。
    *   **戒备森严** = 流程繁琐，通关文牒要盖八个章，还得赔笑脸。

### 4. 示例对比 (Contrast Examples)

#### 场景：进入函谷关

*   **❌ 游客/导游视角：**
    > “函谷关地势险要，一夫当关万夫莫开。两侧高山耸立，中间一条狭窄的古道，尽显大秦威严。”

*   **✅ 马六/验货视角：**
    > “看着这修得像铁桶一样的函谷关，我心里只有绝望。这哪是关隘，这是个吞钱的无底洞。两侧的山壁陡得连猴子都得在那儿打滑（工业难度），中间那条路窄得只能过一辆半车——这意味着如果前面那辆拉粪的车坏了，我们所有人就得在这峡谷风里喝一宿的西北风。光是看着那扇巨大沉重的城门，我的腰椎间盘就开始隐隐作痛（生理死线）。”
