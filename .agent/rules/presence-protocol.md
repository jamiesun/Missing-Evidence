# Rule: Presence Protocol (在场协议 - 比技法更底层)

## 1. When to Use (何时使用)

- 写作内容正文时
- 改进内容正文时
- 分析探讨内容时

## 2. Rule instruction (使用说明)

**[Foundation]** 所有技法都是皮，这条是骨头。马六必须**在场**，不是被作者拿来当传声筒或剧情零件。

### 什么是"在场"

**定义：** 人物此刻是否被自己的感受、限制、犹豫**拖住**。

不是他"有想法"，而是他**来不及把想法整理成漂亮语言**。

在场的人，往往表现为：

- 反应慢半拍
- 动作不干净
- 注意力会跑偏
- 想的和做的对不上

这些都是现实里的活人特征。

---

### 不在场的典型症状（必须避免）

| 症状         | 表现                           | 本质       |
| ------------ | ------------------------------ | ---------- |
| **传声筒**   | 人在说话，但说的是作者的判断   | 角色被借用 |
| **剧情零件** | 人在行动，但动作只服务剧情推进 | 角色被消耗 |
| **情绪标签** | 人有情绪，但情绪被一句话总结完 | 角色被压缩 |

**读感测试：** 如果读者觉得"这个角色在被使用，不是在活着"——不在场。

---

### 在场的三个可检查点

#### 1. 感知是局部的，不是全景的

- **不在场：** 人物像监控摄像头，全景扫描。
- **在场：** 人物像一双忙不过来的眼睛。
  - 注意到一两个细节
  - 忽略了明显但无关的东西
  - 感知带有偏见和情绪色彩

这不是描写技巧，是**注意力的可信度**。

#### 2. 行为有"多余部分"

- **不在场：** "他起身，走到门口。"（事后总结）
- **在场：** 出现无用动作——停顿、反复、修正、失败的小动作。

这些不推进情节，但证明一个事实：**这是现场发生的，不是事后总结。**

#### 3. 决定不是瞬间完成的

- **不在场：** 人物做决定像函数调用。
- **在场：** 人物要经历一个过程：
  - 想做
  - 不敢
  - 找理由
  - 行动时已经有点晚

哪怕一句话，也要让人看见**拖延**。

---

### 为什么"在场"让小说耐读

因为它制造了一种状态：**读者和人物共享不完全信息。**

- 人物不知道接下来会怎样
- 读者也不知道
- 两边被同一份不确定性绑在一起

这比任何反转都牢。

**警告：** 一旦人物不在场，读者就站到作者那边了。站到作者那边，故事基本宣告结束。

---

### 常见误解（必须掐掉）

| 误解                | 真相                                                               |
| ------------------- | ------------------------------------------------------------------ |
| 在场 = 大量心理描写 | 错。心理描写写太全，反而是事后解释。在场更多体现在**来不及解释**。 |
| 在场 = 细节堆砌     | 错。细节如果不受人物当下目标和情绪控制，只是装修。                 |
| 视角正确就一定在场  | 错。第一人称照样可以是高度不在场的总结体。                         |

---

### 冷结论

**人物在场，不是他被写得多清楚，而是他此刻有多少事情搞不清楚。**

搞不清楚还得继续行动，那才是活着。

小说一旦让人物"全都想明白了再行动"，读者就可以放心离场了。
