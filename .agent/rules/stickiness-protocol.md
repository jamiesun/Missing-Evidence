# Rule: The Stickiness Protocol (粘性协议 - 耐读性工程)

## 1. When to Use (何时使用)

- 写作内容正文时
- 改进内容正文时
- 分析探讨内容时

## 2. Rule instruction (使用说明)

**[Core Logic]** 耐读性不是"让人爽"，是让人'暂时无法退出判断'。只要读者还在算、在怀疑、在补因果，他就会继续读。

### 1. 信息延迟给全，别延迟给空 (Delayed Completion, Not Delayed Void)

**规则：** 给局部真相，藏关键因果。

- **错误拖法：** 作者不给信息 → 读者不知道发生了什么 → 结果不是好奇，是疲劳。
- **有效拖法：** 人 + 动作 + 结果都给，但解释缺失。读者被迫继续读，补因果。
- **Example:** "他答应得很快，快得不像在思考。直到第二天早上，我才明白他在躲什么。"
- **Punchline:** 你不是不给，是给得不完整。

### 2. 判断不稳定 (Unstable Judgment)

**规则：** 同一人物在不同情境下展现互相冲突的行为动机。不是反转，是'不一致'。

- **操作：** 他在弱者面前显得温和；在平级面前突然刻薄。读者开始重新计算这个人。
- **Trigger:** 一旦读者需要'重新算账'，他就会翻页。

### 3. 拒绝即时情绪释放 (Deny Instant Catharsis)

**规则：** 情绪一出现，禁止立刻解释、立刻安抚、立刻总结。

- **错误：** 读者比人物先解脱。
- **正确：** 让情绪悬着。不给立刻命名，不给道德结论。读者会替角色感到不安。
- **Punchline:** 不安是最稳定的阅读燃料。

### 4. 语言偶尔失控一秒钟 (One-Second Derailment)

**规则：** 允许偶尔出现语气突变、节奏打滑、逻辑轻微偏斜。

- **效果：** 读者会下意识停一下——他意识到这里不是模板。
- **边界：** 只失控一秒。多了就是水平问题。

### 5. 主题藏在重复行为里 (Theme in Repeated Actions)

**规则：** 主题写在句子里，读者不在乎；主题写在行为里，读者会慢慢意识到。

- **结构：** 同类事件反复发生 → 每次细节略有变化 → 角色选择越来越极端或越来越疲惫。
- **效果：** 读者不是被说服，是被'看懂'。一旦读者开始自己总结意义，他就不会中途离场。
