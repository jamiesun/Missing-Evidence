# Workflow: 番茄书荒推文生成器 (Fanqie Shuhuang Tweet Generator)

---
description: 根据书籍大纲和经典章节生成150-360字整本书推荐推文，提取核心剧情线索，充分利用悬念、情绪钩子、金句与经典剧情组合
---

## 0. 两种模式

本工作流支持两种生成模式：

| 模式 | 输入 | 输出字数 | 适用场景 |
|------|------|---------|---------|
| **章节模式** | 单个章节文件 | 120-140字 | 日更推文、章节预热 |
| **全书模式** | 大纲 + 经典章节 | 150-360字 | 新书推荐、书评区引流 |

---

# Part I: 全书推文模式（重点）

## 1. 输入与准备

### 1.1 必须读取的素材

| 素材类型 | 路径规范 | 用途 |
|---------|---------|------|
| **书籍大纲** | `stories/{书名}/outline/` | 理解整体架构、核心冲突、人物关系 |
| **前5章正文** | `stories/{书名}/chapters/C01-C05` | 开篇氛围、主角出场、世界观建立 |
| **作者标注的经典段落** | 大纲中标注或作者指定 | 金句来源、高光时刻 |

### 1.2 信息提取矩阵

从大纲和章节中识别以下核心要素：

| 要素 | 提取内容 | 示例 |
|------|---------|------|
| **核心困境** | 主角面临的不可能任务/死局 | "现代博士穿越到战国，没有金手指，只有物理公式" |
| **反差设定** | 现代 vs 古代 / 高大上 vs 接地气 | "985材料博士 vs 没有卫生纸的绞肉机时代" |
| **核心武器/卖点** | 主角靠什么活 | "脑子里的力学公式" / "墨家机关" |
| **情绪底色** | 全书的情感基调 | "疲惫的尊严" / "带血的幽默" |
| **一句话定位** | 这本书是什么 | "地狱里的成本核算施工日志" |

---

## 2. 全书推文结构公式

> [!IMPORTANT]
> **字数：150-360字**。不是流水账，是精心设计的"信息陷阱"。

### 2.1 六段式结构

```
【1. 刺痛开场】(20-40字)
   ↳ 一句让读者停下拇指的话，必须有"肉感"或"反常识"
   ↳ 示例："历史书上不会告诉你，乱世的味道，其实是发酵的血水混合着三天没洗的汗臭。"

【2. 人物速写】(30-50字)
   ↳ 主角是谁 + 原本的人生轨迹 + 命运给的"巨大玩笑"
   ↳ 用现代标签建立共鸣（985/房贷/发Paper）
   ↳ 示例："马静仪，某985材料力学在读博士，原本的人生规划是发Paper、毕业、进大厂、还房贷。"

【3. 绝望递进】(40-60字)
   ↳ 用"没有……没有……只有……"的排比，层层剥夺希望
   ↳ 必须包含"生理性威胁"（饥饿/伤口/钝刀）
   ↳ 示例："这里没有系统，没有金手指，没有'叮'的一声签到奖励。只有真实的饥饿、感染的伤口、以及随时可能砍下来的生锈钝刀。"

【4. 反差碰撞】(40-60字)
   ↳ 核心卖点：现代知识 vs 古代困境
   ↳ 用"当……遇到……"的碰撞结构
   ↳ 降维打击不是爽，是"接烂尾项目的倒霉蛋"
   ↳ 示例："当现代物理学撞上古代攻城战，当'杨氏模量'遇到'墨家机关'，她发现自己并不是来拯救世界的，而是一个被迫接手烂尾项目的倒霉项目经理。"

【5. 核心卖点/金句】(30-50字)
   ↳ 主角的"武器"和"KPI"
   ↳ 必须有一句可以单独截图的金句
   ↳ 示例："她唯一的武器，是脑子里的力学公式；她唯一的KPI，是活过今晚。"

【6. 升华收尾】(30-50字)
   ↳ 点题 + 重新定义这本书是什么
   ↳ 用隐喻代替直白描述
   ↳ 示例："这不是一首英雄的赞歌，这是一份关于如何在地狱里进行成本核算的施工日志。"
```

### 2.2 结构变体

根据书的调性，可选用以下变体：

| 变体 | 侧重 | 适用书类 |
|------|-----|---------|
| **苦难浪漫版** | 绝望中的微光 | 虐心/成长向 |
| **职场惊悚版** | 用KPI/成本解构史诗 | 硬核/理工/穿越 |
| **沙雕生存版** | 荒诞感拉满 | 轻松/搞笑/日常 |

---

## 3. 核心技法

### 3.1 悬念工程

| 技法 | 说明 | 示例 |
|------|------|------|
| **信息缺口** | 只说"发生了什么"，不说"怎么解决" | "她被扔进了一个连卫生纸都没有的绞肉机时代" |
| **概念碰撞** | 把两个本不该出现在一起的词放一起 | "杨氏模量 vs 墨家机关" |
| **降维自嘲** | 把主角的优势说成负担 | "被迫接手烂尾项目的倒霉项目经理" |

### 3.2 情绪钩子

| 钩子类型 | 触发点 | 示例 |
|---------|-------|------|
| **共鸣钩** | 读者的日常痛点 | "房贷还没还完" / "还得排三小时队" |
| **恐惧钩** | 具体的生理威胁 | "生锈钝刀" / "感染的伤口" |
| **好奇钩** | 反常识的组合 | "物理学家在尸山血海里搭榫卯" |

### 3.3 金句锻造

**公式**：`[微观的武器] + [宏观的KPI]`

- ✅ "她唯一的武器，是脑子里的力学公式；她唯一的KPI，是活过今晚。"
- ✅ "我不是想活着，我只是还没算完这笔账。"
- ❌ "她凭借智慧逆袭了所有人。"（太爽文）

## 4. 马六风格调味剂

### 4.1 必须包含

| 元素 | 描述 | 检查点 |
|-----|------|--------|
| **脏兮兮的真实感** | 必须有一个让人"闻到味道"的细节 | "发酵的血水" "三天没洗的汗臭" |
| **成本意识** | 把宏大叙事降维成"划不划算" | "成本核算" "烂尾项目" |
| **现代职场隐喻** | 用读者熟悉的语言翻译古代困境 | "985博士" "KPI" "项目经理" |
| **生理性限制** | 主角的身体在拖后腿 | "饥饿" "伤口" "活过今晚" |

### 4.2 禁止事项

- ❌ **剧透核心反转**：只说"有反转"，不说"是什么"
- ❌ **爽文语气**："吊打" "碾压" "无敌" "震惊所有人"
- ❌ **空洞大词**："史诗" "传奇" "震撼人心"（除非用来解构）
- ❌ **系统流暗示**：如果书里没系统，不要暗示有
- ❌ **过度美化主角**：保持"倒霉蛋"底色

### 4.3 情绪调性

- **底色**：黑色幽默 + 疲惫的尊严 + 一丝不死心的倔强
- **语气**：像一个熬夜赶ddl的老社畜在给你推荐ta唯一能看进去的书
- **节奏**：短句打节奏，长句拉情绪

```

    ↓

    ↓

    ↓

    ↓

    ↓
输出推文 + 卖点 + 金句库 + 标签
```

## 7. 质量检查清单（全书模式）

### 7.1 结构检查
- [ ] 开头第一句能让人停下拇指？
- [ ] 有具体的人物标签（职业/学历/身份）？
- [ ] 有"没有……只有……"的绝望递进？
- [ ] 有现代知识 vs 古代困境的碰撞点？
- [ ] 有可单独截图的金句？
- [ ] 结尾用隐喻重新定义了这本书？

### 7.2 马六风格检查
- [ ] 有一个让人"闻到味道"的脏细节？
- [ ] 有成本/KPI/项目管理的隐喻？
- [ ] 主角被描述成"倒霉蛋"而非"天选之人"？
- [ ] 语气像疲惫的老社畜在安利？

### 7.3 禁忌检查
- [ ] 没有剧透核心反转？
- [ ] 没有使用"震惊""竟然""无敌"等烂词？
- [ ] 没有暗示书里有系统/金手指（如果没有的话）？

### 7.4 传播力检查
- [ ] 金句脱离上下文也能理解？
- [ ] 读完有"这书在哪儿看"的冲动？
- [ ] 适合粘贴到评论区/书评区？

# Part II: 章节推文模式（保留原版）

## 8. 输入要求

用户需提供：
- **章节文件路径**：待生成推文的章节 markdown 文件

---

## 9. 阅读与分析章节

### 9.1 提取核心元素
从章节中识别以下要素，**每个要素至少提取2-3个候选项**：

| 要素类型 | 描述 | 用途 |
|---------|------|------|
| **钩子事件** | 最能抓住读者注意力的冲突/危机/反转 | 用于"悬念猎手版" |
| **金句候选** | 角色的精彩对话或叙述者的犀利吐槽 | 用于所有版本的核心锚点 |
| **情绪锚点** | 让读者产生共鸣的情感时刻（无奈、心酸、愤怒、疲惫） | 用于"情绪共振版" |
| **荒诞细节** | 反差感极强的场景（生死与屎尿、宏大与琐碎） | 用于"黑色荒诞版" |
| **画面感段落** | 最有视觉冲击力的描写 | 作为各版本的"加料包" |

### 9.2 素材分类标签
提取后为每个素材打标签，便于差异化组合：

```
[悬念] 刚逃出死人堆，又发现自己在敌国通缉名单上
[情绪] 他说"活着就是赚到"，眼眶红了但没哭
[荒诞] 讨论生死存亡时，主角在计算草鞋的折旧率
[金句] "我不是想活着，我只是还没算完这笔账。"
```

## 10. 三版必输出架构

> [!IMPORTANT]
> **每次生成必须输出至少3个差异化版本**，针对不同读者口味。

### 版本矩阵

| 版本名称 | 核心策略 | 开头侧重 | 目标读者 |
|---------|---------|---------|---------|
| **🎣 悬念猎手版** | 钩子 + 悬念 + 留白 | 制造"然后呢？"的冲动 | 喜欢悬疑/反转的读者 |
| **💔 情绪共振版** | 痛点 + 共鸣 + 情感锚 | 直击打工人/社畜灵魂 | 容易被角色境遇打动的读者 |
| **🤡 黑色荒诞版** | 反差 + 黑色幽默 + 沙雕 | 用荒诞消解苦难 | 喜欢段子手/开心麻花风格的读者 |

---

## 11. 各版本写作指南

### 🎣 版本A：悬念猎手版

**结构公式**：
```
【致命悬念开场】(20-30字) ← 直接抛出"不可能的困境"
【背景快闪】(25-35字) ← 极简交代，只给必要信息
【关键转折暗示】(30-40字) ← 透露一点，藏更多
【断崖式结尾】(20-30字) ← "直到——"然后不说了
【书名+钩子词】(10-15字)
```

**写作要点**：
- 开头必须有**时间压力**或**生死边缘**
- 中间用"我以为……直到……"结构
- 结尾必须是**半句话**，强制读者去找答案
- 金句可用可不用，服从悬念节奏

**示例**：
> 穿越第一天，我就被埋进了尸坑。  
> 更糟糕的是，我发现压在我身上的那具尸体，体温还是热的。  
> 我以为这是地狱。直到我被人从坑里拖出来，看见了那面写着"墨"字的黑旗——  
> 《墨攻：榫卯》

---

### 💔 版本B：情绪共振版

**结构公式**：
```
【情绪共鸣开场】(20-30字) ← 用感受代替事件（累、疼、饿、绝望）
【困境白描】(30-40字) ← 具体的、可感知的苦难细节
【核心金句】(25-35字) ← 必须有一句可以单独截图传播的话
【微光/反抗暗示】(20-30字) ← 不是爽，是"还没死"
【书名+情感词】(10-15字)
```

**写作要点**：
- 开头用**生理感受**（"三天没吃饭""脚趾磨出血泡"）
- 中间可以借用**职场隐喻**（甲方、deadline、KPI）
- 金句必须有，且放在**倒数第二段**
- 结尾用"但"或"所以"转折，暗示不屈

**示例**：
> 饿了三天，我终于弄明白了一件事：  
> 在战国乱世当个普通人，比我读博那五年还累。  
> "我不是想活着，我只是还没算完这笔账。"  
> 所以我接了墨家的差事——修一辆永远修不好的独轮车。  
> 《墨攻：榫卯》，社畜的硬核求生日志

---

### 🤡 版本C：黑色荒诞版

**结构公式**：
```
【荒诞反差开场】(20-30字) ← 把苦难和琐事并列（"刚逃过一劫，发现鞋掉了"）
【吐槽式叙述】(30-40字) ← 用抱怨的语气讲生死大事
【成本核算/职业病】(25-35字) ← "这事儿划不划算"的灵魂拷问
【荒诞升级/神转折】(20-30字) ← 比开头更离谱
【书名+沙雕词】(10-15字)
```

**写作要点**：
- 语气像**吐槽大会**，但内容是生死
- 必须有**成本计算**（"这条命换不换得起三贯钱"）
- 生理细节拉满（屎尿屁、饿、疼、痒）
- 结尾可以用**反问**或**自嘲**

**示例**：
> 刚从尸坑里爬出来，我发现最大的问题不是浑身脓疮，  
> 而是我那双草鞋只剩一只了。  
> 更绝的是，这里的"墨家"居然管饭——但伙食成本要从"贡献值"里扣。  
> 穿越躲过了死，却躲不过996。  
> 《墨攻：榫卯》，一本让你觉得自己加班也没那么惨的书

## 12. 马六风格调味剂（章节版共用）

### 12.1 必须包含（每版至少1项）
| 元素 | 描述 | 示例 |
|-----|------|------|
| **成本意识** | 主角做事要算账，提及代价 | "这条命换得起三贯钱不？" |
| **生理细节** | 具体的身体感受 | "胃里空得像个塌了的馒头" |
| **职业病吐槽** | 现代社畜视角解构古代困境 | "这不就是跨部门协作吗" |

### 12.2 禁止事项
- ❌ 直接剧透关键反转
- ❌ 使用"震惊""竟然""没想到""绝绝子"等烂梗词
- ❌ 纯粹的爽文语气（"吊打""碾压""无敌""戳瞎狗眼"）
- ❌ 单版超过150字
- ❌ 三版内容高度雷同（钩子/金句可复用，但结构和侧重必须不同）

### 12.3 情绪调性
- **底色**：黑色幽默 + 无奈感 + 一丝不死心的倔强
- **语气**：像一个疲惫但清醒的打工人在吐槽自己的项目
- **节奏**：短句为主，偶尔一句长句拉住节奏

## 15. 质量检查清单（章节模式）

### 15.1 单版检查（每版都要过）
- [ ] 字数在120-140之间？
- [ ] 开头3秒能抓住注意力？
- [ ] 有具体的场景/画面感？
- [ ] 没有剧透核心反转？
- [ ] 语气符合"疲惫但清醒的社畜"调性？
- [ ] 结尾有引导点击的钩子？

### 15.2 三版差异化检查
- [ ] 三版开头句式/切入点不同？
- [ ] 三版情绪调性有明显区分？（悬念紧张 vs 情感沉重 vs 荒诞轻快）
- [ ] 同一个金句没有在三版中都用？（最多复用于2版）
- [ ] 三版不能只是换了几个词的"换皮"？

### 15.3 金句独立传播测试
- [ ] 至少有1句金句，脱离上下文也能看懂+有共鸣？
- [ ] 金句不超过25字？（方便截图/评论区引用）
